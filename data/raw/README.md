좋아. 이제 `lexicon_builder.py` 작성에 필요한 정보는 충분합니다. 아래 기준으로 짜면 됩니다.

```text
확정 사항

1. 입력 파일 경로
   - data/raw/vocab_5965.xlsx
   - data/raw/vocab_40k.xlsx

2. 읽을 시트
   - vocab_5965.xlsx: 첫 번째 시트
     pandas 기준 sheet_name=0

   - vocab_40k.xlsx: 여섯 번째 시트
     pandas 기준 sheet_name=5

3. 헤더
   - 두 파일 모두 첫 행이 컬럼명
   - pandas 기준 header=0

4. 기본 사전 우선순위
   - 4만 개 5등급 목록을 메인 기준으로 사용
   - 5965개 목록은 보조 정보로만 사용

5. 5965 등급 처리
   - A → 1등급성 보조값
   - B → 1등급성 보조값
   - C → 2등급성 보조값

   즉, 기존 A/B/C를 3단계 난도로 쓰지 않고,
   A/B는 쉬운 어휘, C는 약간 덜 쉬운 어휘 정도로만 반영

6. 4만 목록 등급 처리
   - 1등급 → 가장 쉬움
   - 5등급 → 가장 어려움
   - 최종 difficulty의 핵심 기준

7. 병합 기준
   - lemma + homograph_no + normalized_pos 기준으로 병합
   - 단, 병합 실패를 줄이기 위해 내부적으로 보조 병합 로직을 넣을 예정

8. 병합 방식
   - outer merge
   - 4만 목록에만 있는 단어도 유지
   - 5965 목록에만 있는 단어도 유지하되 낮은 우선순위로 처리

9. 최종 출력
   - data/processed/lexicon_master.csv
```

난도 계산은 기존보다 이렇게 바꾸는 게 맞습니다.

```text
word_difficulty =
0.80 × 4만 5등급 기반 난도
+ 0.10 × 5965 보조 등급 난도
+ 0.05 × 5965 순위 난도
+ 0.05 × 어종/분야 보정
```

다만 5965 정보가 없으면 4만 기준이 거의 전부가 되도록 재가중합니다.

```text
4만 목록에 있음:
   4만 등급 중심으로 계산

4만 목록에 없고 5965에만 있음:
   5965 등급 + 순위로 임시 난도 계산
   source = vocab_5965_only

둘 다 있음:
   4만 등급을 우선하고 5965는 약한 보정만 적용
   source = both
```

특히 5965의 A/B/C는 신뢰도를 낮게 둘게요. 사용자가 말한 것처럼 이 자료는 “정밀 난도 사전”이라기보다 **기초 어휘 여부를 보조 확인하는 자료**로 보는 게 맞습니다.
